# 대출상담사 소개 — 디자인 가이드

> 카페인 디자인 시스템(`design-system/DESIGN.md`)을 베이스로, 대출상담사 소개 화면에 특화된 컴포넌트·레이아웃·카피 결정을 정리한 문서.
> 본 문서는 **본 프로덕트의 결정사항**만 다룬다. 토큰/타이포/색상 등 시스템 전반은 카페인 DESIGN.md 참조.

---

## 0. 핵심 원칙 (5)

1. **매물이 anchor다.** 위치/지역이 아니라 **매물**을 중심으로 상담사가 매칭된다. "이 단지·이 가격대를 처리할 수 있는 사람이 누구인가"가 본질.
2. **한 화면 한 메시지, 미니멀.** 카페인 톤 그대로 — 큰 한 글자 같은 강한 헤드 + 여백 + 8pt 그리드. 정보를 더 넣고 싶을 때마다 멈추고 한 번 더 뺀다.
3. **카드 컨테이너는 매물에만.** 상담사 리스트는 **행(row) + 구분선** 구조. 흰 배경 위에서 시각적 노이즈를 최소화.
4. **신뢰감은 정보의 양이 아니라 위계에서 온다.** 별점·응답시간·경력 — 3가지만 일관된 위치에 놓는다.
5. **카피는 해요체 + 사실 우선.** "지금 바로!", "추천!" 같은 광고성 표현 금지. 카페인 UX writing 원칙 그대로.

## 1. 정보 모델

### 1.1 매물 (`PROPERTY`) — anchor

| 필드 | 예시 | 노출 |
|---|---|---|
| `complexName` | 미캘란쉐르빌 | 매물 카드 헤드 |
| `unit` | 101동 1502호 | 매물 카드 서브 |
| `priceLabel` | 15억 5,000 | 매물 카드 (만원 suffix) |
| `address` | 서울 강남구 테헤란로 152 | 상세에서만 |
| `ltvLimit` | 108,500만원 | 상세 시뮬레이션에서만 |
| `recommendedBank` | shinhan | 진입 컨텍스트 매칭에 사용 |

**원칙**: 매물 카드는 **4요소만** — 아파트명, 동/호수, 시세, 이미지. 면적·매물유형 등 부수 정보는 절대 카드에 넣지 않는다.

### 1.2 상담사 (`CONSULTANT`)

매물-anchor 모델이라 **매물 기준 매칭 정보**가 1순위.

**카드 노출 필드 (필수 6개)**
- `name` · `org` · `bank` · `career` · `rating`/`reviewCount` · `respondMins`

**상세에서만 노출**
- `regNo` (대출모집인 등록번호) · `consultCount` · `avgRate` · `specs` 전체 · `bio` · 후기 본문

**매칭 메타 (E안 추천순/매물 적합도용)**
- `complexHandled` (이 단지 처리 건수) · `priceRangeHandled` (가격대 처리 건수) · `bankMatch` (1순위 은행 일치) · `matchScore` (종합 0-100) · `matchReason` (한 줄 설명)

### 1.3 진입 컨텍스트 (`ENTRY_CONTEXTS`)

| context | 출처 | 기본 동작 |
|---|---|---|
| `none` | 독립 진입 (매물 없음) | 상담사 중심 리스트 (별도 화면) |
| `loancompare` | 대출비교 1순위 결과 | 매물 + 1순위 은행 가중치 |
| `realestate` | 부동산 매물 상세 | 매물 + 전 은행 평등 |

진입 컨텍스트 라벨은 **카드 안에 넣지 않는다.** 현재는 표시 자체를 보류 — 필요해지면 매물 카드 위 별도 작은 칩으로.

## 2. 컴포넌트

### 2.1 매물 카드 (Property Card)

#### 2.1.1 큰 카드 — 세로 (`PropertyCardLargeV`) — **메인**

리스트 화면 상단에 들어가는 hero. 풀폭 썸네일 위, 정보 아래 구조.

```
┌──────────────────────────────┐
│                              │
│      [썸네일 140px]          │   ← 풀폭, repeating-linear-gradient
│                              │
├──────────────────────────────┤
│ 미캘란쉐르빌    다른 매물 둘러보기 ›│
│ 101동 1502호                  │
│ 15억 5,000만원                │   ← 한 줄, tnum
└──────────────────────────────┘
```

| 속성 | 값 |
|---|---|
| border | 1px solid `#E5E7EB` |
| border-radius | 14px |
| 썸네일 높이 | 140px |
| 썸네일 패턴 | `repeating-linear-gradient(135deg, #F3F4F6 0 12px, #E5E7EB 12px 24px)` |
| 정보 영역 패딩 | 16px |
| 아파트명 | 18px / 800 / `#1F2937` / letter-spacing -0.3px |
| 동·호수 | 12px / 500 / `#6B7280` |
| 시세 | 15px / 700 / `#1F2937` / tnum / 한 줄 |
| "만원" suffix | 13px / 600 / `#6B7280` |
| "다른 매물 둘러보기 ›" | 12px / 700 / `#3182F6`, 우상단 정렬 |

#### 2.1.2 큰 카드 — 가로 (`PropertyCardLargeH`)

좁은 영역 또는 정보 밀도가 필요할 때. 썸네일 좌, 정보 우.

| 속성 | 값 |
|---|---|
| 썸네일 | 72×72, radius 10 |
| 정보 정렬 | flex 1, gap 14 |
| "다른 매물 둘러보기 ›" | 우측 정렬, align-self flex-start |

#### 2.1.3 작은 카드 — sticky용 (`PropertyCardSmall`)

상세 화면 상단 sticky에 들어가는 컴팩트 한 줄.

| 속성 | 값 |
|---|---|
| 배경 | `#FAFAFA` (border 없음) |
| 썸네일 | 40×40, radius 8 |
| 아파트명 | 14px / 800 |
| 동호수 | 11px / 500 / `#9CA3AF`, 인라인 |
| 시세 | 12px / 700 / tnum |
| "다른 매물 ›" | 11.5px / 700 / `#3182F6` |

#### 2.1.4 "다른 매물 둘러보기" — 카피 결정

- ❌ "변경" (딱딱함)
- ❌ "다른 매물 보기" (의도 약함)
- ✅ **"다른 매물 둘러보기"** — 친절·탐색 의도 반영

### 2.2 상담사 행 (`ConsultantRow`) — **메인 E안**

리스트 화면의 기본 단위. **카드 컨테이너 없음 · 흰 배경 · 구분선만**.

```
┌──────────────────────────────────────┐
│ ⊙   김서윤              ›│
│ 신한 신한부동산금융 강남센터 · 경력 12년 │
│     ★ 4.9 (127) · 평균 4분            │
└─────────────────────────────────────┘
   ↑                                ↑
   아바타 52px                    chevron 20px
   + 은행 뱃지 오버레이
```

| 속성 | 값 |
|---|---|
| 행 패딩 | 18px 4px |
| 행 사이 구분선 | 1px solid `#F3F4F6` (첫 행 제외) |
| 아바타 크기 | 52px |
| 이름 | 16px / 800 / letter-spacing -0.3px |
| 메타 (소속·경력) | 13px / 500 / `#9CA3AF` |
| 별점·응답 | 12px / 700 (별점) · 12px / 600 (응답) |
| chevron | 20×20, stroke `#D1D5DB`, stroke-width 2 |
| gap (아바타-텍스트) | 14px |

**원칙**: 행 사이 구분선은 매우 옅게 (`#F3F4F6`) — 시각적 분리는 여백으로, 구분선은 보조.

### 2.3 아바타 + 은행 뱃지 (`CCAvatar`)

**결정의 핵심**: 두 동그라미(프로필 + 은행 로고)를 가로로 나열하면 시각적 혼란. **은행 뱃지를 아바타 우하단에 오버레이**하여 하나의 시각 단위로 묶는다.

```
   ⊙
  김\●  ← 은행 뱃지 (size × 0.40)
```

| 속성 | 값 |
|---|---|
| 아바타 모양 | 원형 |
| 아바타 배경 | `consultant.avatarTone` (각 상담사 고유 톤) |
| 이니셜 | 성 한 글자 / size × 0.4 / 700 / `#1F2937` |
| 뱃지 위치 | `right: -2, bottom: -2` |
| 뱃지 크기 | `Math.round(size * 0.40)` |
| 뱃지 흰 보더 | `padding: 1.5px`, `background: #fff` |
| 뱃지 내부 | 실제 은행 로고 이미지 (없으면 텍스트 fallback) |

**아바타 톤 팔레트** (각 상담사 고유)
- 신한: `#FDE2D7` (피치)
- 하나: `#FFF1C7` (옐로우)
- KB: `#DCE9FE` (블루) — 토스 4단계 중 "여유"
- NH: `#E7F0DE` (그린)
- 우리: `#E7F0DE` (그린)

### 2.4 별점 (`CCStar`)

| 속성 | 값 |
|---|---|
| 별 아이콘 | 12×12, fill `#F59E0B` |
| 점수 | 12px / 700 / `#1F2937` |
| 후기 수 | 12px / 500 / `#9CA3AF`, 괄호 |
| gap | 3px |

`★ 4.9 (127)` 패턴. 점수는 한 자리 소수까지.

### 2.5 은행 로고 (`CCBank` + `CC_BANK_LOGOS`)

```js
const CC_BANK_LOGOS = {
  shinhan: 'assets/banks/shinhan.png',
  // 나머지는 텍스트 fallback
};
```

- 실제 PNG 로고가 있으면 `<img>` 원형 crop
- 없으면 은행별 컬러 원 + 단축명 텍스트 (`신한`, `KB`, `우리` 등)
- KB/카뱅 같은 밝은 배경은 `#1F2937` 텍스트, 나머지는 흰 텍스트

**추가 작업 필요**: 신한 외 은행 로고 (KB, 하나, NH, 우리, 카뱅, 토스) — Common-BI 피그마에서 export 받아 `assets/banks/`에 추가.

### 2.6 상담사 카드 변형 (참고용 — A/B/C/D)

**메인은 E (Row)지만**, 다른 컨텍스트에서 카드형이 필요할 경우 대비.

| 변형 | 추가 정보 |
|---|---|
| A · 미니멀 | 5요소만 (이름·소속·경력·별점) |
| B · + 응답 속도 | "평균 N분 응답" 한 줄 |
| C · + 전문 분야 칩 | 이름 옆 작은 칩 (`주택담보` 등) |
| D · 강점 뱃지 | 우측 filled 뱃지 (`갈아타기 전문`, `베테랑 N년`) |

카드 컨테이너 사용 시:
- background: `#fff`
- border: 1px solid `#E5E7EB`
- border-radius: 14px
- padding: 14px
- 페이지 배경: `#F4F4F6` (카드 분리감 확보)

## 3. 화면 구조

### 3.1 매물 중심 리스트 (`ListScreenRow`) — **메인 화면**

```
┌──────────────────────────────────────┐
│ ‹                                    │ ← 상단 바 (10 16 6)
├──────────────────────────────────────┤
│                                      │
│  [매물 큰 카드 — 세로]                │ ← padding 4 16 0
│                                      │
├──────────────────────────────────────┤
│ 이 매물 상담 가능한 상담사 6명         │ ← 19/800, "6명"만 #3182F6
├──────────────────────────────────────┤
│ ⊙ 김서윤              › │
│   신한 ... · 경력 12년               │
│   ★ 4.9 (127) · 평균 4분            │
├──────────────────────────────────────┤
│ ⊙ 최연수              › │
│   ...                                │
└──────────────────────────────────────┘
```

| 영역 | 패딩 | 비고 |
|---|---|---|
| 화면 배경 | `#fff` | 카페인 톤 |
| 상단 바 | 10 16 6 | 뒤로가기만 |
| 매물 카드 영역 | 4 16 0 | 세로 큰 카드 |
| 타이틀 | 24 20 8 | 19px / 800 |
| 행 리스트 | 0 16 24 | 행 자체 패딩 18 4 |

**타이틀 카피**: "이 매물 상담 가능한 상담사 **N명**"
- 숫자만 `#3182F6` 토스 블루 강조
- "추천 상담사 N명" 같은 광고성 표현 X

### 3.2 상담사 중심 리스트 (미정 — V2)

매물 없이 진입한 경우. **default 타겟 사용자 결정 필요** (주담대 탐색 / 갈아타기 / 카테고리 진입).

미정인 상태에서는 진입을 막거나, "매물부터 입력하면 더 정확합니다" 안내 후 매물 입력 플로우로.

### 3.3 상담사 상세 (미정 — 다음 작업)

상단 sticky에 작은 매물 카드 (`PropertyCardSmall`), 본문에 상담사 정보 + 후기 + 매칭 근거.

## 5. 타이포 결정

카페인 6단계 스케일 내에서 본 프로덕트가 실제로 쓰는 사이즈:

| 위치 | size | weight | letter-spacing | line-height |
|---|---|---|---|---|
| 타이틀 ("이 매물...") | 19px | 800 | -0.3px | 1.3 |
| 매물 아파트명 (큰 V) | 18px | 800 | -0.3px | 1.3 |
| 매물 아파트명 (큰 H) | 17px | 800 | -0.3px | 1.3 |
| 매물 아파트명 (작은) | 14px | 800 | - | 1.4 |
| 상담사 이름 (행) | 16px | 800 | -0.3px | 1.3 |
| 상담사 이름 (카드) | 15.5px | 800 | -0.2px | 1.3 |
| 매물 시세 | 15px | 700 | - | 1.4 |
| 상담사 메타 (행) | 13px | 500 | - | 1.4 |
| 별점·후기수 | 12px | 700 / 500 | - | 1.4 |
| 응답 시간 | 12px | 600 | - | 1.4 |
| 동·호수 | 12px | 500 | - | 1.4 |
| 작은 메타 | 11~11.5px | 500~600 | - | 1.3 |

**weight 4종만** — 500 / 600 / 700 / 800. Regular(400)는 본 프로덕트에서 사용 안 함.

**숫자**: `font-feature-settings: "tnum"` — 시세·점수·금액에 적용 (세로 정렬용).

## 7. 인터랙션 (정의된 것)

지금까지 정의된 인터랙션:
